Loops

Loops are ways of repeating actions or statements in the code for a certain number of times or while a certain condition is true. Loops are used heavily in all types of programming and will be needed to solve the math-based problems presented in this tutorial.



The first type of loop we will look at is called the for loop. Its structure is shown below, and it behaves like "for every ... in ..., do ...".

for ....:
	# BLOCK of code

The for loop is created by writing "for" and then an expression that iterates over something, such as a range of numbers or a list of values (lists will be discussed in the next lesson).
Iterating over a range of numbers is done like so:

 for i in range(0,10):
	print(i)
# other code down here

Here, i is a temporary variable. range(a, b) creates a range of numbers starting at the number a and ending at the greatest number less than b.
In this loop, i starts out with a value of 0. The indented block of code is then executed, printing out 0. Then, the program returns to the line declaring the loop, and i is set to the next value in the range: 1. The indented block is executed again, printing out 1. This process continues until 9 has been printed. Then, the loop is complete and the program continues below it.

So, rather than writing... 

print(0)
print(1)
print(2)
print(3)
print(4)
print(5)
print(6)
print(7)
print(8)
print(9)

...we can write just two lines and accomplish the same thing.

Printing out numbers in a range is far from the most important use of for loops. A better use of it is for doing mathematical operations over a range of numbers:

sum = 0
for i in range(1,10):
	sum = sum + i
print(sum)

In this example, sum is created and set to a value of 0. It will keep track of a sum of numbers through each iteration of the loop.
i starts out with a value of 1. Then, the indented block is executed; sum (currently with a value of 0) is set to a new value of itself plus i (currently equal to 1). Thus, sum gets a new value of 1.
In the next iteration of the loop, i is equal to 2. sum (currently equal to 1) is set to itself plus i...or sum = 1 + 2 = 3.
This continues, with sum increasing by the new value of i in each iteration. At the end, after i becomes 9, sum will be equal to the sum of the numbers 1 through 9, which is 45.

You also have the option to specify the interval (other than 1) by which i increases in each iteration of the loop, using range(A,B,C). For example:

sum = 0
for i in range(0,1001,2):
	sum = sum + i

This code adds up all of the even numbers from 0 to 1000, because i starts as 0 and is increased by 2 until it would become greater than or equal to 1001.



For loops can also be used to iterate over a set of values. This will be discussed in the next lesson.

The next type of loop is called the while loop. It behaves like "while ... is true, do ...". It is written like this:

while condition:
	# BLOCK of code

The condition is written just like conditions in an if, else, or elif statement. Unlike for loops, while loops do not use temporary variables, so any variables used in the condition must be defined before the loop.

While loops continue to excecute the code in the indented block until the condition becomes false, at which point the loop is complete and the program continues below it. Because of this, while loops are useful when you do not know how many times something will need to happen, but you know the conditions for when it should stop happening.

While loops can replicate the functionality of for loops.

One complication of while loops is the possibility of creating infinite loops:

i = 0
while i < 100:
	print(i)

In this example, i will never increase because a line like i = i + 1 was left out. Thus, i will always be less than 100, and the program will endlessly print 1 to the console until the program is halted. Try this in one of the exercises above and you will probably have to close your browser to stop your computer from freezing...

Two other important aspects of loops are the break and continue statements. The former is often used with control flow statements (if, else, elif) to "break out" of a loop before it is complete. For example:

for i in range(1000,0,-1):
    if i**0.5 % 1 == 0:
        print(i)
        break

This code finds the greatest perfect square less than 1,000. It uses a for loop starting at 1,000 and decreasing by 1 per iteration until it reaches 1. The if statement uses the modulus operator to check if the square root of the value of i at each iteration is evenly divisible by 1 (that is, if it is a whole number). If so, then i is a perfect square. However, since the loop starts at 1000, we know we only want to print out the first perfect square we hit, so break allows us to exit the loop once we find that.

Similarly, continue skips to the next iteration in a loop, bypassing all of the code beneath it in a block. This is useful in selectively ignoring or skipping certain numbers in a loop, for example.

for i in range(0,101):
    if i % 2 == 1:
    	continue
    print(i)

In this example, only the even numbers from 0 to 100 are printed out. When i is odd, the if statement is true and continue is executed, thus skipping print(i) and "continuing" in the next iteration.

Prime or Composite

Whole numbers can be classified as prime or composite. A number is prime if its only factors are 1 and itself (that is, only 1 and the number itself can evenly divide into the number). For example, 3 is prime because its only factors are 1 and 3. A number is composite if it has more than two factors. For example, 4 is prime because its factors are 1, 2, and 4. The number 1 is neither prime nor composite.

Collatz Function

The Collatz function of a number n is n/2 if n is even, or 3n+1 if n is odd. The mathematician who named this function, Collatz, conjectured that if you start at any number and repeatedly apply this function, you will always eventually reach 1, at which point there is a cycle.
For example, if you start at 13, which is odd, the next number is 3*13+1 = 40. 40 is even, so the next number is 40/2 = 20. Continuing in this way, 20 → 10 → 5 → 16 → 8 → 4 → 2 → 1. 1 is odd, so it becomes 4, which goes back to 2 and then 1 again. So 13 took 9 iterations of the function to reach 1.
