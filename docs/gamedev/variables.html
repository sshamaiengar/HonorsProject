Variables + Game Initialization

Variables are elements common to all programming languages. They can be described as named containers for storing different types of data, such as numbers, words, objects, booleans, and others. These will be explained more in depth later.


Variables in JavaScript are declared and assigned using the following syntax:

var variableName = value;

variableName is whatever you want to call the variable. This should be something that is somewhat descriptive of what data the variable stores. value is replaced with the data that will be assigned to the variable. For example:

var myAge = 18;
var mySchool = "Jamestown High School";
var canDrive = true;

In this example, all of the variable names describe what each of its values means. Notice that words or text are surrounded by quotes. These values are called strings, like a string of letters or characters. If they were not surrounded by quotes, the program would try to set mySchool equal to the value of some nonexistent variable named Jamestown, causing it to crash. Also, notice that each line ends with a semicolon. As stated before, this is recommended for all lines including statements, and variable declarations/assignments count as statements.

Also, note that variables can be re-assigned (given a new value) after they have been declared. This is done by simply referring to the variable by name (without var) and setting it equal to something. To change my age to 21 I would write:

myAge = 21;

Like the example above, declare and assign several variables of different types. See that you did it correctly by printing them to the console and checking it.
To print a variable to the console, simply place the variable name within the parentheses after console.log

// write your code below // print your variables below

// write your code below // print your variables below

Another important thing to know regarding variables is that operations can be done using them, such as addition or subtraction between numbers and concatenation (the combining of two strings into one) between strings.

Arithmetic is done using the following symbols: + (addition), - (subtraction), * (multiplication), / (division). These operations can be done between numeric variables (like myAge + yourAge), between variables and numeric values (like myAge + 2), and between numeric values (like 2 + 2). Often you will want to assign the result of a mathematical operation to a variable. This is done simply by writing the mathematical expression after the = sign in a normal variable assignment, like so:

var myAge = 18;
myAge = myAge + 2;

In the above example, the original value of myAge (18), is taken and increased by 2, and then re-assigned to itself. If the variable were printed after running the above code, you would see 20 in the console.

The operation on strings called concatenation is done just like addition:

var firstName = "John";
var lastName = "Doe";
var fullName = firstName + " " + lastName;

The above example would result in the value of fullName being John Doe.



An important part of learning how to program in most languages is learning about object-oriented programming. In this paradigm, a programmer can define objects that have properties/attributes—variables that describe the object—as well as methods—actions that the object can do. In JavaScript especially, objects are thought of as prototypes from which individual instances are created.
For example, you could define an object called Dog, give it properties such as name and breed, and give it actions such as bark and rollOver. Then you could create multiple instances of this Dog object, as if you have multiple Dogs. One instance could be a "German Shepherd" named "Rex", and the other a "Siberian Husky" named "Akira". Each dog is unique but shares types of attributes and actions based on the prototype Dog.
For more details about creating and using your own objects in JavaScript, see here.

In these lessons, it is most important that you understand the use of objected-oriented programming. Phaser, the game engine we will begin using in this lesson, is comprised almost entirely of different types of objects which we will use as we build the game.

This is the basic template of any Phaser game. In the first line, a variable called game is created and given a value. When you see the new keyword in JavaScript, that means a new instance of an object is being created. So this variable is being assigned the value of a Phaser.Game object that has certain specifications which are placed inside the parentheses.

The first two numbers are the height and width in pixels of the game window.
Next, is the rendering context, a choice of which technology you want the game to be drawn on the page with. This choice is not important for this game.

The fourth parameter is an empty set of quotes. Within the quotes, you could specify the location on the page where you want the game window to be placed. This is also unimportant for our purposes, so it should be left empty.

Finally, the last parameter is everything within the set of curly braces. The values after the colons, preload, create, and update, correspond to the names of the functions defined in the following lines. This allows the Game object to execute the actions that you specify in order to run the game properly. These functions will be explained in depth later.

There are several steps that must occur while the game is loading, immediately before it can be played. These steps include creating the ball, the paddle, and the bricks, and ensuring that each can behave properly. These steps all occur within the create function because the actions in that function are executed in order to load the game.

First, the physics engine must be initialized. Phaser provides several different types of physics engines, but for our purposes the Arcade physics engine is best. To initialize the physics engine, use this line:

game.physics.startSystem(Phaser.Physics.ARCADE);

Here, game is our original Phaser game object. It has another object within it called physics, from which we can call an action to start the physics system (an specify the system we want within parentheses).

By default, our game will let objects collide with all four sides of the window and bounce back. However, we want the ball to go out of the screen if we fail to hit it along the bottom, so we must disable collisions on the bottom edge using this line:

game.physics.arcade.checkCollision.down = false;

Next, we want to actually create objects we can see on screen. All objects in Phaser (and most game engines for that matter) are called sprites and include a texture, the graphic representing the object, and a body, the actual object that interacts with others in the physics system. In Phaser, it is often easy to generate textures for sprites using built-in functions, rather than having to create images. We will do this for the ball, paddle, and bricks.

The Phaser Graphics module basically acts as a slate that can be painted on. We can create this slate and position at the origin (top left corner) of the screen using this line:

var graphics = game.add.graphics(0, 0);

Then, we can set a fill color (light blue) and draw a circle with radius 20 on this slate:

graphics.beginFill(0x68b0fd, 1);
graphics.drawCircle(400, 550, 20);

We can turn this graphic on the screen into a sprite placed where we want using the following:

ball = game.add.sprite(game.world.centerX, 520, graphics.generateTexture());

Notice how var is not included at the beginning of the line, as we already defined it above in the code. This line takes the content of the current graphical slate and uses it as a texture when creating the ball at the center of the screen, 520 pixels from the top.

We want to follow this same process for creating the paddle and bricks, but first we must clear the graphical slate:

graphics.destroy();

The code to create the paddle sprite is very similar, except for the paddle we draw a rectangle for the texture, color it white, and place the new sprite slightly below (higher Y coordinate than) the ball.

graphics = game.add.graphics(0, 0);
graphics.beginFill(0xFFFFFF, 1);
graphics.drawRect(350, 560, 100, 20);
paddle = game.add.sprite(game.world.centerX,ball.y+20, graphics.generateTexture());
graphics.destroy();

Lastly, to set up the ball to use physics, add the following code. This will give the sprite a physics body, allow it to bounce, collide with the "walls" of the game, broadcast a message when it has gone off the bottom of the screen, and be positioned based on the center of the circle.

ball.anchor.set(0.5);
ball.checkWorldBounds = true;
game.physics.enable(ball, Phaser.Physics.ARCADE);
ball.body.collideWorldBounds = true;
ball.body.bounce.set(1);

Similarly, for the paddle we add the physics body, allow it to collide and bounce, prevent it from moving on its own when the ball hits it, and force it to be positioned around the center of the rectangle.

paddle.anchor.setTo(0.5,0.5);
game.physics.enable(paddle, Phaser.Physics.ARCADE);
paddle.body.collideWorldBounds = true;
paddle.body.bounce.set(1);
paddle.body.immovable = true;
paddle.body.allowGravity = false;
paddle.body.enable = true;
