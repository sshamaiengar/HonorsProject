Layout and Position

CSS properties can be used to affect the way elements are laid out on the webpage, as well as their positions relative to other elements.

In the previous lesson, you learned about the differences between inline and elements. Sometimes, it may be in your interest to make an inline element display on the page like a block element, or vice versa. This can be accomplished using the display property. For example, images are inline elements by default, so they will be placed immediately next to any previous inline elements. Take these Twitter logos:

If you would prefer that these logos show up on separate lines (like block elements), then just add the display property set to the value of block property to their CSS rules:

img {
	display: block;
	width: 100px;
	height: 100px;
}

To make block elements display like inline ones, use the value inline. There are several other possible values, explained here.



The next major property affecting the positions of elements is aptly named position. With this property you can specify the method used to figure out where an element should go. The default value is static, meaning elements appear on the page in the order they are in the HTML code.


Another value for this property is relative. This makes an element appear on the page in a specific position relative to where it would appear normally (with position: static). For example, it could be shifted 20px to the right and 20px down from its normal position.


Another value for this property is absolute. This makes an element appear on the page in a specific position relative to its first parent element (element that contains this one) that is also positioned relatively or absolutely. This value removes an element from the normal flow of the page because it allows you to place an element exactly where you want it. Refer to the following example to see what absolute and relative positioning can do.

CSS (in <style> element)

div {
	position:relative;
	left:100px;
	background-color: black;
	height:100px;
}
p {
	position:absolute;
	background-color: white;
	color:black;
	right:200px;
}
<div>
	<p>This is positioned absolutely.</p>
</div>

This is positioned absolutely.



As you can see, the div is shifted 100px to the left of where it would normally be. Since the div is positioned relatively, the paragraph inside it is positioned 200px to the right of the edge of the div.



The last main value for this property is fixed. This option is self-explanatory. It leaves the element in one position on the screen even while the rest of the page is moved or scrolled around. This is especially useful for things you want to always be visible on the screen, such as navigation bars.


Position: static. Click to toggle. Try scrolling.



In the previous example, you saw the use of two other CSS properties, left and right. Along with top and bottom, these properties are how you describe the position of elements that have the position property set to a value other than static. These properties take values in px (pixels), %, pt (points) and others. Just like with these units for height and width, using percents is better for making your website adaptable to different screen sizes, while using pixels or points will maintain the same position (relative to the properties/sides you use) regardless of screen size.
