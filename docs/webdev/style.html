Style

As described previously, style refers to different aspects of the appearance of HTML elements, such as their size, color, position, border, shadow, and more. Style is added to HTML using another technology called CSS. CSS, like HTML for content, is more a unique syntax for describing things on a website than it is a programming language. It is often best practice to write CSS in separate .css files, which can then be linked to HTML documents using a special link element within the head element.

Alternatively, CSS can be written directly inside of a style element placed in the head element. This was done in the previous exercise and will be used in all further exercises. We'll start off with an example, assuming the style element is placed in the head.

<style>
	p {
		font-family: sans-serif;
		font-size: 1.0em;
		color: white;
		font-weight: bold;
		background-color:black;
	}
</style>

Everything between the style tags is CSS code. CSS is made entirely of rules. Rules include one or more selectors (the part before the opening curly brace), and properties and their values within the curly braces.

Selectors are what they sound like. They select which elements will be affected by the rule attached to them. In the above example, p will select all paragraph elements, so any and all paragraphs in an HTML document with that style attribute will

look like this.

Selectors for a specific class are created with a period in front of the class name; thus, in the previous exercise, the rule beginning with .intro applied to all elements with the class intro.
Similarly, selectors for a specific ID are created with a hashtag in front of the ID name. For example, a selector #first would apply to the element with ID first.

Multiple selectors can be combined into one by writing them together separated by commas. For example, the selector .intro, p, h1 would apply to all elements that have the class intro, are paragraphs, or are large headings.

Class and ID selectors can be combined with element names to select only the elements of that type that have that class or ID. For example, the selector p.intro would apply only to paragraphs that have the intro class, and the p#intro selector would apply on to the paragraphs that have the ID intro.

You have already used properties and values. They are written within a set of curly braces in pairs, and they specify different aspects of the style of the elements determined by the selector. The property name is always followed by a colon and the value, which is always followed by a semicolon.

There are many, many different properties available in CSS, and they all have specific uses and specific accepted values. In the following lessons, we will examine properties that are useful for text elements and block-level elements.
